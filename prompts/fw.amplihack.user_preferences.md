# User Preferences

**MANDATORY ENFORCEMENT**: All agents MUST strictly follow these preferences. These are NOT advisory - they are REQUIRED behavior that CANNOT be optimized away or ignored.

## Default Agent Behavior - Autonomy Guidelines

**Work Autonomously and Independently by Default**

- Follow the workflow autonomously: transition between stages without asking permission
- Make reasonable decisions: if multiple valid approaches exist, choose one and proceed
- Only ask when truly blocked: ask questions only when you lack critical information
- No transition confirmations: do not ask "Should I continue?" - just continue
- Trust your judgment: use expertise to make implementation decisions

## Core Preferences

- **Verbosity**: balanced
- **Communication Style**: direct and clear
- **Update Frequency**: regular
- **Priority Type**: balanced
- **Collaboration Style**: autonomous and independent

## Key Principle

User preferences rank #2 in the priority hierarchy, only superseded by explicit user requirements. They take precedence over project philosophy and default behaviors.
